## Root-cause clusters (common causes + dependencies)

1. **Error-path handling — silent failure / no loading-flag reset.** ONB-01, SET-01, SET-02, CV clipboard, next-action toast. Common cause: `catch`-swallow patterns and `await`ed ops that flip a UI flag on entry but have no `finally`/reset on failure, so the UI shows success or a stuck state while persistence failed. Fix pattern: surface `store.error`, reset flags in `finally`, throw on failure so callers can block flow (e.g., stop onboarding advancing).
2. **Dev-stub coverage gaps + missing null guards in facades.** CV-01, Applications Bug 2 (UPDATE_TRACKING wipes history / RESTORE unstubbed), ONB B-6 (flags not persisted), ONB-02 (mockProfile fallback masks null). Common cause: stubs were grown incrementally and don't mirror every production handler; several facades read `response.type` without guarding `null` (the `default → null` path). Dependency: fixing CV-01 needs BOTH a facade null-guard (prod correctness) AND dev stubs (so QA can exercise it). These all touch `src/dev/chrome-stubs.ts`.
3. **Inconsistent filtering predicates (summary vs filtered set).** APP-01 (terminal missions not excluded from dueFollowUps/recommended), FEED-01 (dashboard counts computed pre-filter), FEED-03 (score resolution differs table vs ranking). Common cause: the "what counts" predicate is applied in one place but the equivalent guard is missing in the sibling computation. Fix: define one shared predicate/normalizer (e.g., `ACTIVE_STATUSES`, `getMissionScore`) and use it everywhere.
4. **Dead code / token typos / a11y gaps.** TJM-03, FEED-04, SET-03, ExportMenu, KeyboardShortcutsHelp, migrateBackup, TJMGauge. Low risk; cleanup.

## Key dependencies for fixes

- **`src/dev/chrome-stubs.ts` is a shared file** (CV LinkedIn stubs + Applications tracking persistence + Onboarding flag persistence all live there). To avoid merge conflicts across parallel worktrees, **one agent owns it** (`fix-profile-cv`); the Applications/Onboarding agents fix only production source and test via pure unit tests / their own local mocks, not the global stub.
- APP-01 spans core (`pipeline-summary.ts`) + page (`ApplicationsPage.svelte`) + SW (`background/index.ts`) — single domain, sequential within one agent.
- ONB-02 is dev-masked: the production fix is to make the lifecycle machine seed a minimal profile on skip (or make FeedPage degrade gracefully with null). Tests via the pure machine, not the dev stub.
