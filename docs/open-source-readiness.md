## Status

MissionPulse is prepared for a public repository with the following baseline:

- MIT license file present at the repository root.
- Root and workspace package metadata declare the MIT license.
- Local `.env` files, private keys, packaged extension ZIPs, and generated browser test output are ignored.
- Current tracked files were scanned for common secret patterns.
- Personal profile URLs were replaced with neutral examples.
- GitHub community files live in `.github/`.
- Release artifacts are generated by CI instead of committed.

## Maintainer Checklist Before Switching Repository Visibility

- Confirm GitHub repository settings: branch protection, required CI checks, and security advisories.
- Configure optional secrets only in GitHub Actions: `CODECOV_TOKEN`, `CHROME_EXTENSION_ID`, `CHROME_CLIENT_ID`, `CHROME_CLIENT_SECRET`, `CHROME_REFRESH_TOKEN`.
- Review tracked `audits/` and `openspec/` files for product-sensitive context before public launch.
- Replace placeholder privacy contact details in [privacy-policy.md](./privacy-policy.md).
- Run `pnpm ci:check` locally and confirm GitHub Actions pass on a fresh pull request.

## Secret Audit Commands

```bash
git grep -n -I -i -E "(password[[:space:]]*[:=]|api[_-]?key[[:space:]]*[:=]|secret[[:space:]]*[:=]|token[[:space:]]*[:=])"
git ls-files | rg '(^|/)\\.(env|env\\.)|\\.zip$|\\.pem$|\\.key$|\\.p12$|\\.mobileprovision$'
```

The first command has false positives for environment variable names and GitHub Actions secrets. Review matches before treating them as findings.
