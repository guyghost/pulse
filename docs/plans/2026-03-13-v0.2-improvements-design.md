# MissionPulse v0.2 — Design

Date: 2026-03-13

## Objectif

Avancer sur 3 fronts en parallele : features manquantes, UX/ergonomie, distribution Chrome Web Store.

## Chantier A — Features manquantes

### Connecteur Malt
- Parser HTML malt.fr dans `src/lib/core/connectors/malt-parser.ts`
- Connector shell dans `src/lib/shell/connectors/malt.connector.ts`
- Detection session via chrome.cookies
- Extraction : title, client, description, stack, tjm, location, remote, duration

### Connecteur Comet
- Parser HTML app.comet.co dans `src/lib/core/connectors/comet-parser.ts`
- Connector shell dans `src/lib/shell/connectors/comet.connector.ts`
- Meme pattern que Free-Work/Malt

### TJM Intelligence (sans LLM)
- Cablage du tjmMachine existant avec l'aggregateur pur
- Affichage : mediane, percentiles, tendance
- Integration LLM reportee

### Settings Page
- Toggles connecteurs (enable/disable)
- Intervalle de scan (slider)
- Edition profil (reprend le form onboarding)
- Reset storage

### Connector Panel
- Statut par connecteur (connected/disconnected/error)
- Dernier sync timestamp
- Refresh manuel par connecteur

## Chantier B — UX / Ergonomie

### Notifications push
- `chrome.notifications` apres scan quand nouvelles missions avec score >= 70
- Permission `notifications` deja dans le manifest

### Tri du feed
- Dropdown dans le header : par score (defaut), date, TJM
- Extension du feedMachine

### Filtres avances
- Par source (Free-Work, Malt, Comet)
- Par remote type (full, partial, onsite)
- Par TJM min/max
- Par stack (chips selectionnables)
- Extension du filtersMachine

### Empty states
- Feed vide (pas encore de scan)
- Aucun resultat (filtres trop restrictifs)
- Erreur connecteur (session expiree, etc.)

### Error handling UI
- Toast/snackbar pour erreurs de scraping
- Message "session expiree" avec lien vers la plateforme

## Chantier C — Distribution

### Assets graphiques
- Logo SVG final
- Screenshots du sidepanel (feed, TJM, settings)
- Banniere promotionnelle 1280x800

### Privacy policy
- Page statique (GitHub Pages ou markdown)
- Donnees collectees, stockage local uniquement, pas de tracking

### Listing Chrome Web Store
- Description FR
- Categorie : Productivity
- Justification des permissions

### Landing page
- One-pager : features, install, screenshots
- Hebergement GitHub Pages

### Compte developpeur
- Inscription Chrome Web Store (25$)
- Premier upload

## Hors scope (YAGNI)

- Backend / proxy LLM
- Analytics / telemetry
- Systeme de comptes utilisateur
- i18n (francais uniquement)
- Import/export de donnees

## Architecture

Aucun changement structurel. Integration dans l'architecture FC&IS existante :
- Parsers purs → `src/lib/core/connectors/`
- Connectors shell → `src/lib/shell/connectors/`
- TJM → cablage tjmMachine + aggregateur existant
- Settings → nouvelle page Svelte + chrome.storage
- Notifications → `src/background/index.ts` (post-scan)
- Filtres/tri → extension feedMachine + filtersMachine
